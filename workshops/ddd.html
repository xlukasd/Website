Mastering DDD for Scalable Systems

This talk explores the concept of Domain-Driven Design (DDD) and its application in software development. The talk will delve into the differences between Anemic Domain Model (ADM) and Rich Domain Model (RDM) and how they shape system scalability, maintainability, and overall quality. The speaker will discuss the building blocks of a Rich Domain Model, such as Entity, Value Objects, and Aggregate Root, and how they help build a ubiquitous language between developers and stakeholders using a vocabulary that describes the business rules. The talk will also cover the importance of context boundaries in DDD and how they help define the entities, value objects, and aggregates that model the domain.
